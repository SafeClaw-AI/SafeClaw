# tests/contracts/

> 基于 `specs/` 自动推断的合同测试集合。

## 当前覆盖

当前合同测试已经覆盖以下几类稳定约束：

- `specs/` 基本元数据完整性
- Worker 状态机的稳定标识与关键终态约束
- Effect Ledger 的追加式历史与三账本提交协议关键点
- Preflight 的严格降级 / 只升不降规则
- 错误码注册表的唯一性与分类完整性
- SPI 注册表与公开扩展点一致性
- `generated/` 最小索引产物与当前 `specs/` 同步

## 与其他门禁的关系

- `tests/contracts/`：做协议级断言
- `tools/checks/`：做一致性 / 版本 / 结构 / 文档 / 骨架 / 同步检查
- `tools/lint/`：做稳定命名约束检查

## 当前目标

当前不是为了堆测试数量，
而是为了把 **未来 AI 自动生成、自动修复、自动扩展时最容易漂移的地方** 先锁死。
