# M2 产品价值优先级改序决议

- Time: 2026-04-01 00:48:36 +0800
- Status: historical external-integration sequencing snapshot
- Scope: preserve the old sequencing snapshot for future external integration/fusion reference

## 0. 边界说明（2026-04-04）
- SafeClaw 当前不单独开发丞相模式/大都督模式等外部解释层功能。
- 本文档仅保留历史改序快照与后期拼接融合参考，不作为 SafeClaw 当前主线功能承诺。
- 若后续需要接入，只做外部程序拼接融合，不在 SafeClaw 仓内继续扩写独立模式功能。

## 1. 为什么现在改序
- 现有 `M2-1` / `M2-2` 已经证明：面板人话层可以做，但它还不足以向非技术用户证明 SafeClaw 的核心价值。
- 当前真正缺的不是再多一层解释，而是“做完一件真实事情后，用户能看到账单、能撤销、能截图给别人看懂”的产品证据。
- 因此本轮把 `M2` 从“先解释系统”改为“先证明价值”，后续再继续补解释层、入口层、模型层。

## 2. 三道墙（冻结口径）
| 墙 | 当前问题 | 当前结论 |
| --- | --- | --- |
| 语言墙 | JSON / 状态机 / 错误码对普通人不友好 | 先做人话账单、人话结论、人话撤销承诺 |
| 入口墙 | 终端仍是主要实操入口 | 暂不跳 GUI；先把价值证明做出来，再进入 Web UI |
| 模型墙 | 真实大众用户不会自己配模型/API key | 暂不假装已解决；先继续沿本地/维护层边界推进 |

## 3. M2 新顺序（冻结版）
| 顺序 | 刀 | 目标 | 验收标准 |
| --- | --- | --- | --- |
| 1 | M2-P0-1 | Effect Ledger 可读账单 | 任一真实任务执行后，能输出普通人可读的操作账单 |
| 2 | M2-P0-2 | 一个真实任务场景 | 至少 1 个真实任务可跑通、可截图、可复现 |
| 3 | M2-P0-3 | `undo` 一键撤销 | 能撤销上一次真实任务造成的全部副作用 |
| 4 | M2-P0-4 | 产品价值闭环验收 | 跑真实任务 → 看账单 → 执行 `undo` → 完全还原 |
| 5 | M2-P1-1 | `丞相检查` 最小检查 | 把最小检查链翻译成人话，不把普通用户扔回终端 |
| 6 | M2-P1-2 | `丞相版本` 真源回显 | 明确版本号、真源、镜像状态 |
| 7 | M2-P1-3 | `丞相验板` 固定验收单 | 给普通用户稳定、短、可执行的人工验收步骤 |
| 8 | M2-P1-4 | `丞相修复` 维护层边界 | 明确哪些能自动修，哪些必须提示或停下确认 |

## 4. 什么保持不变
- 普通用户主入口仍是 `官方 Codex 面板`。
- 终端仍只保留给维护、安装、迁移、排障使用。
- `docs/chancellor-mode/v2/02-m2-panel-command-truth-source.md` 仍是四个丞相命令的字段真源；只是优先级临时后置，不再抢在产品价值证明之前。
- 已完成的 `M2-1` / `M2-2` 不作废，视为后续解释层的已完成地基。

## 5. 当前停线规则
- 在 `M2-P0-4` 完成前，不跳去做 Web UI / Electron / Tauri 主线开发。
- 在 `M2-P0-4` 完成前，不把“接真实模型”当成当前主线交付物。
- 若出现会让现有毕业链失真的高风险问题，仍优先回到 `M1` 门禁稳定性处理。

## 6. 当前阶段验收标准
- 至少 1 个真实任务可以被普通人看懂地执行完。
- 执行结束后能看到一张普通人能读懂的操作账单。
- 执行 `undo` 后，副作用能被完整还原。
- 这条完整路径能被截图发给非技术用户，对方能理解发生了什么。
