# SafeClaw Roadmap

## 中文

以下路线图代表当前方向，后续可能会根据用户反馈持续调整。

### Phase 0 — Brand Foundation
- 建立 GitHub 仓库主页与基础品牌信息
- 发布首个公开版本 `v0.0.1`
- 确认官方域名与统一商标声明

### Phase 1 — Public-Facing Basics
- 完善 README、愿景文档与公开路线图
- 准备官网占位页
- 补充最基础的文档、演示与下载入口说明

### Phase 2 — Safer Execution Experience
- 提供更清晰的执行说明与风险提示
- 设计更友好的确认、预览与恢复流程
- 让用户更容易理解自动化与 AI 的行为边界

### Phase 3 — User-Centered Product Direction
- 根据真实用户反馈调整产品定位
- 优化首次使用体验
- 让非技术用户也能更轻松地上手

## English

This roadmap reflects the current direction and may evolve with real user feedback.

### Phase 0 — Brand Foundation
- Set up the GitHub repository presence and core brand information
- Publish the first public release: `v0.0.1`
- Establish official domains and a consistent trademark statement

### Phase 1 — Public-Facing Basics
- Improve the README, vision document, and public roadmap
- Prepare a simple website placeholder
- Add lightweight documentation, demos, and download guidance

### Phase 2 — Safer Execution Experience
- Provide clearer execution guidance and risk communication
- Design more user-friendly confirmation, preview, and recovery flows
- Help users better understand the boundaries of automation and AI behavior

### Phase 3 — User-Centered Product Direction
- Refine the product direction based on real user feedback
- Improve the first-use experience
- Make SafeClaw easier to approach for non-technical users
