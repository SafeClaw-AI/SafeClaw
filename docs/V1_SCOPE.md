# SafeClaw V1 范围与规划

## 核心原则
**SafeClaw V1 不是平台，也不是全能 Agent。它就是一个有龙虾灵魂、默认谨慎、本地可控、离线可用、开箱即用的自动化执行工具。**

---

## 一页结论

| 项目 | 结论 |
| --- | --- |
| V1 唯一定位 | 有龙虾灵魂的、本地可控、离线可用、开箱即用的自动化执行工具 |
| V1 核心卖点 | 真诚提醒风险、默认不乱动、用户说了算、过程全透明 |
| V1 开发策略 | 不推倒重来，只打磨已跑通 MVP，做减法，不扩张 |
| V1 成功标准 | 用户第一次打开就能理解、敢用、能停、能查、能回放 |
| V1 禁区 | 不做平台化，不做复杂编排，不做大而全，不做炫技功能 |

---

## 产品定位

### V1 对外一句话
**SafeClaw 是一个有龙虾灵魂的本地自动化执行工具：先讲风险，再讲操作，最后由你决定是否执行。**

### V1 对外三句介绍
| 场景 | 表达 |
| --- | --- |
| 用户第一次认识你 | SafeClaw 帮你把本地自动化任务跑起来，但不会擅自替你做决定。 |
| 用户最该感知的差异 | 它默认谨慎、默认可查、默认可停，不是黑箱式乱执行。 |
| 用户为什么愿意用 | 它开箱即用，离线可跑，所有关键权限都握在用户手里。 |

### 品牌核心气质
| 维度 | 要求 |
| --- | --- |
| 语气 | 真诚、克制、像站在用户这边的朋友 |
| 行为 | 不抢决定权、不故作聪明、不装万能 |
| 原则 | 风险先说清，权限先收紧，动作可撤回，过程可追溯 |

---

## V1 功能边界

### 必须保留并极致打磨
这些功能已经是 V1 主体，不新增赛道，只把体验打透：

| 模块 | V1 要求 |
| --- | --- |
| 流程录制 | 稳定录、稳定存、稳定复现 |
| 键鼠回放 | `dry-run` 与真实执行一致，状态清晰 |
| 本地任务执行 | PowerShell / CMD / Bash 跑得稳，失败可解释 |
| 自然语言任务预览 | 从“描述动作”升级为“先说风险，再讲步骤，最后等确认” |
| 一键取消 | 随时可停，停止反馈明确 |
| 安全控制 | 默认全关，用户手动开启，开启前明确解释影响 |
| Web Doctor | 真能排障，不只是展示信息 |
| 审计与回放 | 用户能看懂每一步、每次权限、每次结果 |
| 一键启动 / 一键离线启动 | 首次使用简单，不让用户先配半天环境 |

### V1 必补的灵魂核心项
这些是“差异化灵魂”，开发量不一定大，但必须有：

| 项目 | 本质 |
| --- | --- |
| 底层安全红线锁 | 永远先提示风险，必须确认才执行，绝不替用户做决定 |
| 原生龙虾灵魂话术 | 所有预览、告警、确认、失败提示都统一人格 |
| 实时心跳状态条 | 永远告诉用户“我正在做什么” |
| 本地基础记忆系统 | 只存本地、可查看、可删除、可清空 |

---

## 明确冻结项

这些不是“不好”，而是 **V1 不该碰**。现在碰，只会把节奏拖乱。

| 冻结项 | 原因 |
| --- | --- |
| OpenClaw 迁移兼容 | 对首发用户不是刚需，且容易把产品叙事带偏 |
| 人机协同、桌面流、Skill 体系 | 会把产品带向平台化和复杂化 |
| 复杂流程编排、可视化编辑 | 开发量大、维护成本高、与 V1 核心不一致 |
| 灵魂自定义、预设角色库 | 不是第一版差异化关键，容易分散注意力 |
| 高级参数配置 | 会伤害开箱即用体验 |
| 本地大模型适配、云端 API 对接、龙虾搜索 | 第二版再说，V1 先把本地离线闭环打死 |

---

## V1 的四条底层原则

建议把这四条写进产品文案、任务预览逻辑、确认逻辑、测试用例，作为全局标准：

| 原则 | 解释 |
| --- | --- |
| `Risk First` | 每次执行前先讲风险，再讲收益和步骤 |
| `Consent First` | 关键动作必须用户明确确认 |
| `Local First` | 默认本地、默认离线、默认不出站 |
| `Visible First` | 全程可见、可查、可停、可回放 |

---

## 验收标准

V1 不按“功能数量”验收，而按下面 6 条验收：

| 验收项 | 标准 |
| --- | --- |
| 好懂 | 新用户 3 分钟内知道这工具是干什么的 |
| 敢用 | 用户知道它不会偷偷执行 |
| 可控 | 默认全关，关键权限必须手动开 |
| 可停 | 执行中随时能取消 |
| 可查 | 每一步都能看见、能回放、能排障 |
| 有魂 | 所有交互都体现“先说风险、你来决定” |

---

## 版本路线

### V1
先把 **“本地自动化执行 + 风险预览 + 用户确认 + 全流程透明”** 做成一个真正可信的小而美产品。

### V2
等 V1 发布后，基于真实用户反馈，再决定是否恢复：
- 迁移兼容
- Skill 体系
- 桌面流
- 可视化编排
- 云端/模型扩展能力
