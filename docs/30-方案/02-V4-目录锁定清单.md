# V4 目录锁定清单

## 目的

- 本文件是当前仓库的目录锁定依据。
- `docs/reference/01-反屎山工程规范.md` 中涉及“目录是否可改”的判断，统一以本文件为准。
- 后续开发先按本文件执行；若需要新增、迁移、删除目录，必须先更新本文件，再做实际改动。

## 当前执行原则

1. 先锁结构，再做开发。
2. 没有写进本清单的目录，不默认允许新增。
3. 已存在但不理想的历史路径，先标为“迁移前临时保留”，不得继续扩张同类路径。
4. 结构整改与业务切片分开推进，不混做一刀。

## 根目录锁定

### 一、长期保留的根目录
- `.github/`：仓库协作与 CI 配置
- `assets/`：长期资产文件
- `config/`：配置真源
- `docs/`：正式文档区
- `generated/`：当前生成产物
- `image/`：图片资产
- `modules/`：模块代码
- `safeclaw-core/`：核心实现
- `safeclaw-sqlite/`：SQLite 相关实现
- `specs/`：协议真源
- `temp/`：人工清理过程文件与历史停放区
- `tests/`：测试与合同校验
- `tmp/`：临时验证输出目录
- `tools/`：检查、生成与维护工具

### 二、迁移前临时保留的根目录
- `01_文档/`：历史中文方案文档；迁移前保留，不新增同类历史目录
- `target/`：运行/构建产物目录；仅作运行态与构建输出，不作为真源文档或源码落点

### 三、长期保留的根文件
- `README.md`
- `STATUS.md`
- `CHANGELOG.md`
- `DECISIONS.md`
- `ARCHITECTURE.md`
- `.gitignore`
- `LICENSE`
- `VERSION`
- `Cargo.toml`
- `Cargo.lock`
- `safeclaw.cmd`
- `safeclaw.ps1`

### 四、迁移前临时保留的根文件
- `MVP_PROGRESS.md`：现有主台账；迁移前保留，不新增同类根台账
- `PUSH_LOG.md`：现有推送流水；迁移前保留，不新增同类根流水账
- `开发计划.md`：现有推进面板；迁移前保留，不新增同类根计划文件

## 后续整改约束

### 立即生效
- 不再新增新的根目录台账种类。
- 不再默认向 `docs/round_logs/` 追加新的轮次留痕。
- 新的结构整改文件统一落到 `docs/30-方案/`。

### 单独立项后再做
- 根目录三份台账迁移到更合规的正式位置。
- `docs/round_logs/` 历史留痕整体迁移。
- `tools/checks/`、公开文档、历史引用一并改到新路径。

## 变更规则

- 若要新增目录、删除目录、迁移目录，必须先修改本文件并说明理由。
- 若本文件与历史习惯冲突，以本文件为准；历史路径按“迁移前临时保留”解释，不视为继续扩张的依据。
- 若后续形成新的正式目录结构，更新本文件时必须同步更新 `docs/README.md` 与相关检查脚本。
