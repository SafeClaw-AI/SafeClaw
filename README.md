# SafeClaw

> 一只为普通用户准备的「安心龙虾」。
>
> 简单、透明、可控，让自动化和技术不再让人紧张。

---

## 它是什么

SafeClaw 是一个面向普通用户的本地自动化执行系统。

它不是黑箱助手，也不是只为开发者准备的复杂工具箱。
它更像一只站在你这边的「安全龙虾」：

- 先讲风险，再讲步骤，最后由你决定是否执行
- 默认谨慎、默认可查、默认可停、默认不越界
- 把复杂留给系统，把轻松留给用户

我们的目标不是让用户学习技术，
而是让用户 **不需要懂太多原理，也能安心使用自动化能力**。

---

## 它想解决什么问题

很多自动化工具很强，但普通人不敢用：

- 看不懂它到底要干什么
- 不知道哪里有风险
- 一旦点了执行，怕出事后收不回来
- 遇到权限、错误、脚本、命令行就容易紧张

SafeClaw 想做的，是把这些不安感降到最低：

- **透明**：做什么、为什么做、风险在哪，都说清楚
- **可控**：关键动作必须由你决定
- **可恢复**：错误、超时、中断之后，尽量能恢复、回滚、补偿
- **本地优先**：尽量在你自己的设备上完成，不把核心控制权交出去

---

## 和 OpenClaw 的区别

**OpenClaw 更像是给开发者的工具箱。**

**SafeClaw 更像是给普通用户的「安心龙虾」。**

两者方向不同，没有冲突：

- OpenClaw 偏开发者效率、能力扩展、工程玩法
- SafeClaw 偏普通用户可理解、可确认、可恢复、可放心使用

SafeClaw 关心的不是“能不能做更多”，
而是“普通人敢不敢放心点下去”。

## 当前开源内容

当前仓库优先公开 SafeClaw 的 **协议层 / 真源层**，用于：

- 稳定代码生成
- 合同测试
- 多模型协作
- 后续 Phase 1 的最小实现

目前公开的核心内容包括：

- `specs/state-machines/worker_lifecycle.json`：Worker 生命周期状态机
- `specs/schemas/effect_ledger.json`：副作用账本与三账本提交协议
- `specs/schemas/action_tiers.json`：Tier × Reversibility 正交定义
- `specs/error-codes/sys_errors.json`：系统错误码注册表
- `specs/config/preflight.json`：预检阈值、降级与运行时纠偏规则
- `specs/spi/base_fields.json`：4 类 SPI 的统一基础字段
- `VERSION`：当前协议层版本

当前 **暂不公开** 的主要是内部架构哲学、设计推导、决策过程与商业化路线相关文档。

## 愿景

让每个人都能放心使用自动化与技术。

## 协议层说明

`specs/` 是当前公开仓库中的单一真源。

它会逐步驱动：

- 合同测试
- 一致性检查
- 版本检查
- 结构完整性检查
- codegen 入口
- 后续的 Rust / Python / TypeScript 生成层

如果你希望参与贡献，请优先从 `specs/` 和未来的 `tests/contracts/` 开始，而不是直接假设内部实现。
